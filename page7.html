Open World System

GitHub

Description

An open world system developed in C++ using the DirectX11 framework for the Advanced Technology third-year module of BSc Games Technology.

The system handles the streaming of geometry data of open world environments to reduce that amount of data that is stored in memory at any time.

Specifications

● Developed in C++ using the DirectX11 framework
● Binary serialization for the loading and unloading of data to storage
● A* pathfinding algorithm

Serialization

The system was designed to serialize geometry and position data into binary files to allow for data to be loaded and unloaded from memory.

Binary Serialization
Binary was chosen as the desired file format for data streaming due to its faster speeds which is important in this system as there is a priority on efficiency when loading and unloading chunks as the user moves around the environment. 

Chunk Loading

The objects in the scene are separated into chunks to allow for the dynamic loading and unloading of geometry data from memory.
As the player moves out of range of a chunk, its geometry data is removed by shutting down the vertex and index buffers for each of the objects within the chunk.
When the player moves back in range of a chunk the geometry data for the objects within the chunk is read from the binary file, and then used to create new buffers for the objects.

Third Person Controller

Through the development of a third person camera system I was able to implement a third person player controller to allow the user to navigate between the chunks making up the environment.
This involves calculating the position of the camera's matrix based on an adjustable algorithm. 

Pathfinding

The NPCs in the scene make use of an A* path-finding algorithm to allow them to navigate throughout the scene.
